# Bug Report: `_was_original_host` uses the wrong source of truth — any reconnecting user can steal host

**Type**: Implementation drift (code doesn't match spec) + Data model gap (missing `created_by` field)
**Severity**: High
**Feature**: 001-lobby-room-management
**Reported**: 2026-06-23
**Status**: Patched
**Patched**: 2026-06-23

## Description

`RoomConsumer._was_original_host()` (`apps/room/consumers.py:241-242`) defines "original host" as:

```python
def _was_original_host(self, room: Room) -> bool:
    return room.host_id != self.user.id and room.host_id is not None
```

This checks "am I not the current host?" — it returns `True` for **every** reconnecting user who isn't the current host, not just the actual original host (room creator).

**Reproduction scenario**:
1. User A creates room → host = A
2. User B joins via link
3. User A disconnects → host transfers to B (correct, per FR-010)
4. User C joins via link
5. User C disconnects briefly, then reconnects
6. `_was_original_host(C)` → `room.host_id (B) != C.id` → `True` → C "regains" host
7. **C is now host despite never being the original host — host stolen by any reconnecting user**

The spec is unambiguous. FR-010 (spec.md:140): "If the **original host** reconnects, they regain the host role immediately." US2/AC7 (spec.md:47): "Given a room where the **original host** reconnects after a disconnection, When they rejoin, Then they immediately regain the host role and control privileges." Both refer to the *original* host — the creator — not "whoever isn't the current host."

## Root Cause Analysis

The `host` field on `Room` is mutable (it changes during host transfer). The original host regain logic needs an immutable reference to the room creator. When `_was_original_host` was implemented, the check was written as `room.host_id != self.user.id` — a proxy for "someone else is host, so I must be the original" — but this conflates "original host" with "not the current host." The data model never included a `created_by` field because the spec's entity definitions (spec.md:170-174) only listed `host` as a key attribute without distinguishing between the original creator and the current host.

## Recommended Fix

1. Add `created_by = models.ForeignKey(User, ...)` to `Room` model (non-nullable, set once, related_name='created_rooms')
2. Generate migration
3. Update `RoomService.create_room()` to set `created_by=host` alongside `host=host`
4. Rewrite `RoomConsumer._was_original_host()` to compare `self.user.id` against `room.created_by_id` instead of `room.host_id`
5. Re-open T044 in tasks.md and add a new task for the `created_by` field
