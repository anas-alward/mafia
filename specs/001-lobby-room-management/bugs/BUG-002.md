# Bug Report: Room model missing `members` reverse accessor; `participant_ids` wrongly stored on Room instead of RoomMember

**Type**: Implementation drift (code diverges from data model intent)
**Severity**: Medium
**Feature**: 001-lobby-room-management
**Reported**: 2026-06-23
**Status**: Patched
**Patched**: 2026-06-23

## Description

Two related issues in `apps/room/models.py`:

### 1. Missing `members` reverse relationship on Room

The `RoomMember.room` FK has no `related_name`, forcing all access through the Django default `room.roommember_set`. The data-model.md explicitly intended `members | M2M → User | through=RoomMember`, meaning `room.members` should be the accessor. Currently:

- `apps/room/models.py:56` — `self.roommember_set.count()`
- `apps/room/models.py:59` — `self.roommember_set.count() >= self.max_members`
- `apps/room/views.py:147` — `room.roommember_set.filter(user=request.user).exists()`

All of these should use `room.members` instead.

### 2. `participant_ids` stored on Room instead of RoomMember

`Room.participant_ids` (JSONField, `default=dict`) maps `user_id → cloudflare_participant_id`. This is per-member data, not room-level data. The field should live on `RoomMember` as a simple `CharField` (e.g., `cloudflare_participant_id`), mirroring the relationship structure. Currently:

- `apps/room/models.py:32` — field defined on Room
- `apps/room/views.py:34` — writes `room.participant_ids[str(room.host_id)] = ...`
- `apps/room/views.py:35` — `room.save(update_fields=['participant_ids'])`

## Root Cause Analysis

When the M2M through model was replaced with a standalone FK model (due to migration constraints), the `related_name='members'` was never added to `RoomMember.room`. The `participant_ids` was placed on Room during the initial Cloudflare Realtime Kit integration because it was the simplest place at the time — the field wasn't reconsidered when RoomMember was refactored.

## Recommended Fix

1. Add `related_name='members'` to `RoomMember.room` FK
2. Add `cloudflare_participant_id = models.CharField(max_length=128, blank=True)` to `RoomMember`
3. Remove `participant_ids` from Room model
4. Update `_create_cloudflare_meeting()` in views.py to write participant ID to the host's RoomMember record instead of Room
5. Update all `roommember_set` references to `members` in models.py, views.py, consumers.py, and services.py
6. Generate migration(s) for both changes
